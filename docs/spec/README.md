# Agent Specs

Behavior contracts scoped to something a team (or agent) can ship and validate independently. Each spec defines external behavior (inputs, outputs, preconditions, constraints) precisely enough for an agent to create an implementation plan from.

Specs are organized by lifecycle phase in subdirectories:

- `Draft/` — specification in progress.
- `Review/` — ready for stakeholder review.
- `Approved/` — reviewed and approved for implementation.
- `Implemented/` — implementation complete and verified.
- `Deprecated/` — no longer active or relevant.

See [list-specs.md](list-specs.md) for the full lifecycle index.
