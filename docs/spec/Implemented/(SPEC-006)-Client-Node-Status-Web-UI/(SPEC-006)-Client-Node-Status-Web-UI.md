## Problem Statement

When WireGuard tunnel issues arise on a remote machine, the operator needs a
quick way to check tunnel state and restart the interface without requiring
SSH access. Family members are not expected to use the CLI. A lightweight
local web page accessible on the LAN provides this without any installed
client software.

## External Behavior

After this spec is implemented:

1. Each provisioned client node runs a local HTTP server on port 8888.
2. Any device on the same LAN can open `http://<lan-ip>:8888/` and see:
   - Peer name and assigned WireGuard IP
   - DNS name (e.g. `laptop.wg`)
   - WireGuard tunnel status (up/down)
   - Last handshake timestamp
   - Transfer bytes (sent/received)
   - Hub endpoint
3. A "Restart Tunnel" button POSTs to `/restart` and triggers
   `systemctl restart wg-quick@wg0` (Linux) or the equivalent launchctl
   command (macOS).
4. The page auto-refreshes every 30 seconds.
5. The server runs as a systemd service (Linux) or LaunchDaemon (macOS) and
   starts automatically on boot.

## Implementation Approach

- Single-file Python script using stdlib `http.server` — no dependencies.
- Peer name and DNS name are embedded at generation time (from `porthole gen-peer-scripts`); tunnel status is read live from `wg show wg0 dump`.
- Server listens on `0.0.0.0:8888`. No authentication (LAN-only service).
- The script is generated alongside the watchdog and tunnel service files by
  `porthole gen-peer-scripts <name>`.
- Generates: `wg-status-server.py`, `wg-status-server.service`,
  `wg-status-server.plist`.

## Constraints

- Python 3.8+ (available on all supported platforms without extra install).
- No third-party packages — must work from a fresh OS install.
- Must not require root to run (reads `wg show` via sudo, writes nothing).
- Port 8888 is fixed; no config file needed for initial MVP.

## Success Criteria

- `porthole gen-peer-scripts <name>` produces `wg-status-server.py` and
  service files.
- Server starts via `systemctl start wg-status-server` and responds at
  `http://localhost:8888/`.
- Status page shows correct peer info and live handshake time.
- Restart button triggers WireGuard restart and returns a confirmation page.
- Auto-refresh works (page reloads every 30 s without user action).
